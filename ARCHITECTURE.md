# 21miron — целевая архитектура

## Физический сервер

Хостовая ОС:

```text
Ubuntu Server 24.04 LTS
Без GUI
Статический LAN IP
Администрирование только с отдельного доверенного ПК
```

На хосте работают:

```text
Nginx
React frontend
FastAPI / Uvicorn
PostgreSQL
Docker
OvenMediaEngine container
SSH
```

На этом же железе, но в отдельной виртуальной машине:

```text
FreeIPA
LDAP/LDAPS
DNS FreeIPA при необходимости
```

FreeIPA не размещается в контейнере OvenMediaEngine и не использует базу данных видеоплатформы.

## Облачный VPS

На VPS остаются существующие сервисы:

```text
WireGuard/VPN
Zabbix
Grafana
BookStack
Draw.io
прочие облачные сервисы
```

VPN-серверы находятся в облаке. Маршрутизация к физическому видеосерверу и FreeIPA выполняется через MikroTik.

## MikroTik

MikroTik является сетевой границей:

- Firewall между VLAN/подсетями;
- проброс только необходимых портов;
- маршрутизация VPN-подсетей;
- ограничение административного доступа;
- запрет прямого доступа к PostgreSQL и Uvicorn;
- при необходимости NAT для RTMP, WebRTC и веб-интерфейса.

Linux-хост не заменяет MikroTik, но локальные сервисы всё равно должны слушать только нужные интерфейсы. PostgreSQL и FastAPI остаются на `127.0.0.1`.

## Роли FreeIPA

```text
video-admins
video-operator
video-viewer
```

### video-admins

- просмотр трансляций;
- панель администратора;
- производительность сервера;
- диагностика;
- управление привязками и конфигурацией.

### video-operator

- только запуск собственной трансляции;
- OBS или браузерный захват экрана/окна;
- нет доступа к просмотру чужих потоков;
- нет доступа к системным метрикам.

### video-viewer

- просмотр разрешённых трансляций;
- оперативная сводка;
- нет административных данных.

## Потоки

Основной ingest:

```text
RTMP -> OvenMediaEngine
```

Воспроизведение:

```text
WebRTC -> основной режим
LL-HLS -> резервный режим
```

Запись потоков на сервер не используется.

## Сетевая схема

```text
Интернет / облачные VPN
          |
       MikroTik
          |
  +-------+--------------------+
  |                            |
  |                     Административный ПК
  |
Физический сервер
  |-- Ubuntu Server host
  |   |-- Nginx
  |   |-- FastAPI
  |   |-- PostgreSQL
  |   `-- OvenMediaEngine
  |
  `-- VM FreeIPA
```

## Миграция со старого MediaMTX

Старый сервер остаётся в работе до завершения тестов нового сервера.

1. Новый сервер получает отдельный IP.
2. Тестовые источники отправляются на новый RTMP URL.
3. Проверяются WebRTC, LL-HLS, нагрузка, роли и маршрутизация.
4. Источники переводятся по одному.
5. Старый MediaMTX остаётся rollback-вариантом до официальной приёмки.
